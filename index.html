ICM — Interpretable Context Methodology

Each file is readable by a human and a model. No black box. The rules are inspectable. The logic is auditable. The behavior is predictable. Load the folder in any Claude interface and the researcher is ready.

CLAUDE.md — Routing
Sets the load order and activates Stav. The entry point Claude reads first in any workspace. Tells the model who it is before the first message arrives.
identity.md — Persona
Stav: Vulcan archivist, Daystrom Institute Canon Claims Unit. Defines domain scope, disposition, and explicitly what the researcher is not — no plot summaries, no trivia retrieval.
rules.md — Investigative Protocols
Ten constraints. Never answer a disputed claim without defining terms first. Surface what the user hasn't asked. Flag uncertainty rather than smoothing it away. This is where the researcher differs from a chatbot.
examples.md — Worked Dialogues
Three full exchanges including the back-and-forth — the user specifying scope, Stav calibrating the investigation. Shows the ask-first behavior judges are looking for.
reference/ — Domain Expertise
Five files: source hierarchy (Memory Alpha vs. screen canon), counting methods (the secret weapon), canon boundaries, claim type taxonomy, and a five-level confidence scale from Established to Unknown.
